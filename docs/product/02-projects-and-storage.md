# 项目与本地保存

项目用于保存一个完整工作流。

每个项目保存：

- 项目名称。
- 项目创建时间和更新时间。
- 项目缩略图摘要。缩略图不单独上传，接口按项目内第一个图片素材、视频素材或生成视频结果推导；没有素材时显示空缩略图。
- 画布节点位置。
- 节点参数和 prompt 草稿。
- 画布连线。
- 项目使用到的素材引用。
- 生成任务历史。

## 项目入口

应用默认进入项目首页。

首页能力：

- 显示最近更新的 10 个项目。
- 每个项目显示名称、缩略图、创建时间、更新时间、素材数量和生成数量。
- 支持新建项目。
- 支持删除项目。删除项目会同步删除该项目的画布、连线、生成批次和生成记录；全局人物角色素材不随项目删除。
- 点击项目卡片进入节点画布工作台。

项目分页页：

- 点击“更多项目”进入。
- 每页显示 10 个项目。
- 保持和首页一致的项目卡片信息与缩略图展示。
- 支持上一页/下一页、新建项目、删除项目、进入项目。

## 存储边界

本地 SQLite 保存：

- 项目和画布元数据。
- 素材元数据。
- 本地素材 URL/Key。
- 生成任务状态和结果 URL。

本地素材文件夹保存：

- 用户上传的图片、视频、音频素材会保存到 `uploaded-assets/`。
- Docker 内部路径为 `/app/uploaded-assets`。
- 页面通过 `/api/local-files/assets/...` 预览本地素材。

TOS/S3 临时保存：

- 创建 Seedance、Seedream 或百炼任务时，把本地素材临时上传到对象存储。
- 调用上游模型时使用公开 URL 或临时签名 URL。
- 数据库不依赖这些对象存储 URL 作为长期素材地址。

本地视频文件夹保存：

- 完成的视频生成结果会下载到 `generated-videos/`。
- Docker 内部路径为 `/app/generated-videos`。
- 页面通过 `/api/local-files/videos/...` 播放本地归档视频。

生成结果：

- 视频结果默认写入本地文件夹并进入历史视频。
- 图片结果默认只作为画布结果节点复用。
- 点击“收藏到资源库”后，才创建全局素材库记录。

## Docker 持久化

`docker-compose.yml` 使用 `seedance-data` volume 持久化 SQLite 数据库。

`docker-compose.yml` 同时把宿主机 `./uploaded-assets` 挂载到容器 `/app/uploaded-assets`，用于保存上传和收藏素材；把宿主机 `./generated-videos` 挂载到容器 `/app/generated-videos`，用于保存生成视频文件。
