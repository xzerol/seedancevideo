# 生成节点

## Seedream 5.0 生图节点

参数：

- Prompt。
- 输入素材引用。
- 比例：智能、16:9、9:16、1:1、4:3、3:4。
- 尺寸：2K、3K。
- 数量：1、2、4 张。

输出：

- 图片结果节点。
- 图片结果会自动保存为本地素材，并进入当前项目素材库。
- 可继续连线到下游 Seedream/Seedance 节点。
- 历史遗留的远程图片结果仍可手动收藏到资源库。

## Seedance 2.0 视频节点

参数：

- Prompt。
- 输入素材引用。
- 模式：参考生成、首尾帧、参考视频。
- 视频比例：智能、16:9、9:16、1:1、4:3、3:4。
- 分辨率：480p、720p、1080p。
- 时长：4、5、6、8、10、15 秒。
- 数量：1、2、4 条。

输出：

- 视频结果节点。
- 可继续作为下游输入源。

状态：

- 提交后节点会显示排队/生成/完成/失败状态。
- 节点内进度条展示当前批次的整体进展。
- 多条视频生成时，下方小圆点对应每条任务：完成为绿色，失败为红色，处理中为灰色。
- 前端会自动轮询本地批次接口，Seedance 返回视频地址后会自动生成或更新右侧视频结果节点。

## 百炼视频节点

模式：

- 文生视频：HappyHorse `happyhorse-1.0-t2v`。
- 首帧图生视频：HappyHorse `happyhorse-1.0-i2v`，需要且只能使用 1 张首帧图片。
- 参考图生视频：HappyHorse `happyhorse-1.0-r2v`，需要 1-9 张参考图。
- 首尾帧：Wan `wan2.7-i2v`，需要且只能使用 2 张图片，分别作为首帧和尾帧。
- 参考视频编辑：HappyHorse `happyhorse-1.0-video-edit`，需要 1 个视频素材，可选 0-5 张参考图。

能力确认：

- HappyHorse `happyhorse-1.0-i2v` 官方接口是首帧图生视频，`media` 中 `first_frame` 要求 exactly one，不是首尾帧接口。
- HappyHorse `happyhorse-1.0-video-edit` 支持输入 1 个 `video` 和 0-5 个 `reference_image` 做视频编辑/参考视频生成。
- 首尾帧能力使用 Wan 首尾帧 API 映射，当前默认模型为 `wan2.7-i2v`，要求 2 张图片。

结果：

- 百炼视频任务进入统一历史视频区。
- DashScope 临时视频地址会转存到 TOS 后再保存，避免过期。

## 对话版本

- 每个生成节点都有“保存对话”。
- 保存内容包括 prompt、引用素材和当前参数。
- “历史版本”可以恢复当前节点的旧对话，不影响其他节点。

## 额度提醒

点击“生成图片”或“生成视频”会调用真实火山方舟或百炼 API，可能产生费用。
